20 /*
21  * ==============================================================================
22  *
23  * FCVROOT Interface Package
24  *
25  * The FCVROOT interface package allows programs written in FORTRAN to
26  * use the rootfinding feature of the CVODE solver module.
27  *
28  * The user-callable functions constituting the FCVROOT package are the
29  * following: FCVROOTINIT, FCVROOTINFO, and FCVROOTFREE. The corresponding
30  * CVODE subroutine called by each interface function is given below.
31  *
32  * ----------------- -----------------------
33  * | FCVROOT routine | | CVODE function called |
34  * ----------------- -----------------------
35  * FCVROOTINIT -> CVodeRootInit
36  * FCVROOTINFO -> CVodeGetRootInfo
37  * FCVROOTFREE -> CVodeRootInit
38  *
39  * FCVROOTFN is a user-supplied subroutine defining the functions whose
40  * roots are sought.
41  *
42  * ==============================================================================
43  *
44  * Usage of the FCVROOT Interface Package
45  *
46  * 1. In order to use the rootfinding feature of the CVODE package the user must
47  * define the following subroutine:
48  *
49  * SUBROUTINE FCVROOTFN (T, Y, G, IPAR, RPAR, IER)
50  * DIMENSION Y(*), G(*), IPAR(*), RPAR(*)
51  *
52  * The arguments are:
53  * T = independent variable value t [input]
54  * Y = dependent variable vector y [input]
55  * G = function values g(t,y) [output]
56  * IPAR, RPAR = user (long int and realtype) data [input/output]
57  * IER = return flag (0 for success, a non-zero value if an error occurred.)
58  *
59  * 2. After calling FCVMALLOC but prior to calling FCVODE, the user must
60  * allocate and initialize memory for the FCVROOT module by making the
61  * following call:
62  *
63  * CALL FCVROOTINIT (NRTFN, IER)
64  *
65  * The arguments are:
66  * NRTFN = total number of root functions [input]
67  * IER = return completion flag (0 = success, -1 = CVODE memory NULL and
68  * -11 memory allocation error) [output]
69  *
70  * 3. After calling FCVODE, to see whether a root was found, test the FCVODE
71  * return flag IER. The value IER = 2 means one or more roots were found.
72  *
73  * 4. If a root was found, and if NRTFN > 1, then to determine which root
74  * functions G(*) were found to have a root, make the following call:
75  * CALL FCVROOTINFO (NRTFN, INFO, IER)
76  * The arguments are:
77  * NRTFN = total number of root functions [input]
78  * INFO = integer array of length NRTFN, with values 0 or 1 [output]
79  * For i = 1,...,NRTFN, G(i) was found to have a root if INFO(i) = 1.
80  * IER = completion flag (0 = success, negative = failure)
81  *
82  * 5. The total number of calls made to the root function (FCVROOTFN), NGE,
83  * can be obtained from IOUT(12).
84  *
85  * If the FCVODE/CVODE memory block is reinitialized to solve a different
86  * problem via a call to FCVREINIT, then the counter variable NGE is cleared
87  * (reset to zero).
88  *
89  * 6. To free the memory resources allocated by a prior call to FCVROOTINIT make
90  * the following call:
91  * CALL FCVROOTFREE
92  * See the CVODE documentation for additional information.
93  *
94  * ==============================================================================
95  */
