21 /*
22  * =============================================================================
23  *
24  * FCVODE Interface Package
25  *
26  * The FCVODE Interface Package is a package of C functions which support
27  * the use of the CVODE solver, for the solution of ODE systems
28  * dy/dt = f(t,y), in a mixed Fortran/C setting. While CVODE is written
29  * in C, it is assumed here that the user's calling program and
30  * user-supplied problem-defining routines are written in Fortran.
31  * This package provides the necessary interface to CVODE for both the
32  * serial and the parallel NVECTOR implementations.
33  *
34  * A summary of the user-callable functions, with the corresponding
35  * CVODE functions, are as follows:
36  *
37  * Fortran CVODE
38  * --------------------- --------------------------------
39  * FNVINITS N_VNew_Serial
40  * FNVINITP N_VNew_Parallel
41  * FNVINITOMP N_VNew_OpenMP
42  * FNVINITPTS N_VNew_Pthreads
43  *
44  * FSUNBANDMATINIT SUNBandMatrix
45  * FSUNDENSEMATINIT SUNDenseMatrix
46  * FSUNSPARSEMATINIT SUNSparseMatrix
47  *
48  * FSUNBANDLINSOLINIT SUNBandLinearSolver
49  * FSUNDENSELINSOLINIT SUNDenseLinearSolver
50  * FSUNKLUINIT SUNKLU
51  * FSUNKLUREINIT SUNKLUReinit
52  * FSUNLAPACKBANDINIT SUNLapackBand
53  * FSUNLAPACKDENSEINIT SUNLapackDense
54  * FSUNPCGINIT SUNPCG
55  * FSUNSPBCGSINIT SUNSPBCGS
56  * FSUNSPFGMRINIT SUNSPFGMR
57  * FSUNSPGMRINIT SUNSPGMR
58  * FSUNSPTFQMRINIT SUNSPTFQMR
59  * FSUNSUPERLUMTINIT SUNSuperLUMT
60  *
61  * FCVMALLOC CVodeCreate, CVodeSetUserData,
62  * and CVodeInit
63  * FCVREINIT CVReInit
64  * FCVSETIIN CVodeSet* (integer arguments)
65  * FCVSETRIN CVodeSet* (real arguments)
66  * FCVSETVIN CVodeSet* (vector arguments)
67  * FCVEWTSET CVodeWFtolerances
68  *
69  * FCVLSINIT CVodeSetLinearSolver
70  * FCVLSSETEPSLIN CVodeSetEpsLin
71  * FCVLSSETJAC CVodeSetJacTimes
72  * FCVLSSETPREC CVodeSetPreconditioner
73  * FCVDENSESETJAC CVodeSetJacFn
74  * FCVBANDSETJAC CVodeSetJacFn
75  * FCVSPARSESETJAC CVodeSetJacFn
76  *
77  * FCVDIAG CVDiag
78  *
79  * FCVNLSINIT CVSetNonlinearSolver
80  *
81  * FCVODE CVode, CVodeGet*, and CV*Get*
82  * FCVDKY CVodeGetDky
83  *
84  * FCVGETERRWEIGHTS CVodeGetErrWeights
85  * FCVGETESTLOCALERR CVodeGetEstLocalErrors
86  *
87  * FCVFREE CVodeFree
88  * --------------------- --------------------------------
89  *
90  * The user-supplied functions, each listed with the corresponding interface
91  * function which calls it (and its type within CVODE), are as follows:
92  *
93  * Fortran: Interface Fcn: CVODE Type:
94  * ------------- ------------------ -----------------------
95  * FCVFUN FCVf CVRhsFn
96  * FCVDJAC FCVDenseJac CVLsJacFn
97  * FCVBJAC FCVBandJac CVLsJacFn
98  * FCVSPJAC FCVSparseJac CVLsJacFn
99  * FCVPSET FCVPSet CVLsPrecSetupFn
100  * FCVPSOL FCVPSol CVLsPrecSolveFn
101  * FCVJTSETUP FCVJTSetup CVLsJacTimesSetupFn
102  * FCVJTIMES FCVJtimes CVLsJacTimesVecFn
103  * FCVEWT FCVEwtSet CVEwtFn
104  * ------------- ------------------ -----------------------
105  *
106  * In contrast to the case of direct use of CVODE, and of most Fortran ODE
107  * solvers, the names of all user-supplied routines here are fixed, in
108  * order to maximize portability for the resulting mixed-language program.
109  *
110  * Important note on portability.
111  * In this package, the names of the interface functions, and the names of
112  * the Fortran user routines called by them, appear as dummy names
113  * which are mapped to actual values by a series of definitions, in this
114  * and other header files.
115  *
116  * =============================================================================
117  *
118  * Usage of the FCVODE Interface Package
119  *
120  * The usage of FCVODE requires calls to a variety of interface
121  * functions, depending on the method options selected, and one or more
122  * user-supplied routines which define the problem to be solved. These
123  * function calls and user routines are summarized separately below.
124  *
125  * Some details are omitted, and the user is referred to the user documents
126  * on CVODE for more complete documentation. Information on the
127  * arguments of any given user-callable interface routine, or of a given
128  * user-supplied function called by an interface function, can be found in
129  * the documentation on the corresponding function in the CVODE package.
130  *
131  * The number labels on the instructions below end with s for instructions
132  * that are specific to use with the serial/OpenMP/PThreads package; similarly
133  * those that end with p are specific to use with the N_VParallel package.
134  *
135  * -----------------------------------------------------------------------------
136  *
137  * Data Types
138  *
139  * Throughout this documentation, we will refer to data types according to
140  * their usage in SUNDIALS. The equivalent types to these may vary,
141  * depending on your computer architecture and on how SUNDIALS was compiled.
142  * A Fortran user should take care that all arguments passed through this
143  * Fortran/C interface are declared of the appropriate type.
144  *
145  * Integers: SUNDIALS uses 'int', 'long int' and 'sunindextype' types. At
146  * compilation, SUNDIALS allows the configuration of the 'index' type, that
147  * accepts values of 32-bit signed and 64-bit signed. This choice dictates
148  * the size of a SUNDIALS 'sunindextype' variable.
149  * int -- equivalent to an INTEGER or INTEGER*4 in Fortran
150  * long int -- equivalent to an INTEGER*8 in Fortran (Linux/UNIX/OSX), or
151  * equivalent to an INTEGER in Windows
152  * sunindextype -- this will depend on the SUNDIALS configuration:
153  * 32-bit -- equivalent to an INTEGER or INTEGER*4 in Fortran
154  * 64-bit -- equivalent to an INTEGER*8 in Fortran
155  *
156  * Real numbers: At compilation, SUNDIALS allows the configuration option
157  * '--with-precision', that accepts values of 'single', 'double' or
158  * 'extended' (the default is 'double'). This choice dictates the size of a
159  * SUNDIALS 'realtype' variable. The corresponding Fortran types for these
160  * 'realtype' sizes are:
161  * single -- equivalent to a REAL or REAL*4 in Fortran
162  * double -- equivalent to a DOUBLE PRECISION or REAL*8 in Fortran
163  * extended -- equivalent to a REAL*16 in Fortran
164  *
165  * -----------------------------------------------------------------------------
166  *
167  * (1) User-supplied right-hand side routine: FCVFUN
168  *
169  * The user must in all cases supply the following Fortran routine
170  *
171  * SUBROUTINE FCVFUN (T, Y, YDOT, IPAR, RPAR, IER)
172  *
173  * It must set the YDOT array to f(t,y), the right-hand side of the ODE
174  * system, as function of T = t and the array Y = y.
175  *
176  * The arguments are:
177  * Y -- array containing state variables [realtype, input]
178  * YDOT -- array containing state derivatives [realtype, output]
179  * IPAR -- array containing integer user data that was passed to
180  * FCVMALLOC [long int, input]
181  * RPAR -- array containing real user data that was passed to
182  * FCVMALLOC [realtype, input]
183  * IER -- return flag [int, output]:
184  * 0 if successful,
185  * >0 if a recoverable error occurred,
186  * <0 if an unrecoverable error ocurred.
187  *
188  * (2s) Optional user-supplied dense Jacobian approximation routine: FCVDJAC
189  *
190  * As an option when using the DENSE or LAPACKDENSE linear solvers, the user may
191  * supply a routine that computes a dense approximation of the system Jacobian
192  * J = df/dy. If supplied, it must have the following form:
193  *
194  * SUBROUTINE FCVDJAC(NEQ, T, Y, FY, DJAC, H, IPAR, RPAR, WK1, WK2, WK3, IER)
195  *
196  * Typically this routine will use only NEQ, T, Y, and DJAC. It must compute
197  * the Jacobian and store it columnwise in DJAC.
198  *
199  * The arguments are:
200  * NEQ -- number of rows in the matrix [long int, input]
201  * T -- current time [realtype, input]
202  * Y -- array containing state variables [realtype, input]
203  * FY -- array containing state derivatives [realtype, input]
204  * DJAC -- 2D array containing the jacobian entries [realtype of size
205  * (NEQ,NEQ), output]
206  * H -- current step size [realtype, input]
207  * IPAR -- array containing integer user data that was passed to
208  * FCVMALLOC [long int, input]
209  * RPAR -- array containing real user data that was passed to
210  * FCVMALLOC [realtype, input]
211  * WK* -- array containing temporary workspace of same size as Y
212  * [realtype, input]
213  * IER -- return flag [int, output]:
214  * 0 if successful,
215  * >0 if a recoverable error occurred,
216  * <0 if an unrecoverable error ocurred.
217  *
218  * (2s) Optional user-supplied band Jacobian approximation routine: FCVBJAC
219  *
220  * As an option when using the BAND or LAPACKBAND linear solvers, the user
221  * may supply a routine that computes a band approximation of the system
222  * Jacobian J = df/dy. If supplied, it must have the following form:
223  *
224  * SUBROUTINE FCVBJAC(NEQ, MU, ML, MDIM, T, Y, FY, BJAC, H,
225  * 1 IPAR, RPAR, WK1, WK2, WK3, IER)
226  *
227  * Typically this routine will use only NEQ, MU, ML, T, Y, and BJAC.
228  * It must load the MDIM by N array BJAC with the Jacobian matrix at the
229  * current (t,y) in band form. Store in BJAC(k,j) the Jacobian element J(i,j)
230  * with k = i - j + MU + 1 (k = 1 ... ML+MU+1) and j = 1 ... N.
231  *
232  * The arguments are:
233  * NEQ -- number of rows in the matrix [long int, input]
234  * MU -- upper half-bandwidth of the matrix [long int, input]
235  * ML -- lower half-bandwidth of the matrix [long int, input]
236  * MDIM -- leading dimension of BJAC array [long int, input]
237  * T -- current time [realtype, input]
238  * Y -- array containing state variables [realtype, input]
239  * FY -- array containing state derivatives [realtype, input]
240  * BJAC -- 2D array containing the jacobian entries [realtype of size
241  * (MDIM,NEQ), output]
242  * H -- current step size [realtype, input]
243  * IPAR -- array containing integer user data that was passed to
244  * FCVMALLOC [long int, input]
245  * RPAR -- array containing real user data that was passed to
246  * FCVMALLOC [realtype, input]
247  * WK* -- array containing temporary workspace of same size as Y
248  * [realtype, input]
249  * IER -- return flag [int, output]:
250  * 0 if successful,
251  * >0 if a recoverable error occurred,
252  * <0 if an unrecoverable error ocurred.
253  *
254  *
255  * (2s) User-supplied sparse Jacobian approximation routine: FCVSPJAC
256  *
257  * Required when using the KLU or SuperLUMT linear solvers, the
258  * user must supply a routine that computes a compressed-sparse-column [or
259  * compressed-sparse-row] approximation of the system Jacobian
260  * J = dfi(t,y)/dy. If supplied, it must have the following form:
261  *
262  * SUBROUTINE FCVSPJAC(T, Y, FY, N, NNZ, JDATA, JRVALS, JCPTRS,
263  * 1 H, IPAR, RPAR, WK1, WK2, WK3, IER)
264  *
265  * This routine must load the N by N compressed sparse column [or row] matrix
266  * with storage for NNZ nonzeros, stored in the arrays JDATA (nonzero
267  * values), JRVALS (row [or column] indices for each nonzero), JCOLPTRS (indices
268  * for start of each column [or row]), with the Jacobian matrix at the current
269  * (t,y) in CSC [or CSR] form (see sunmatrix_sparse.h for more information).
270  *
271  * The arguments are:
272  * T -- current time [realtype, input]
273  * Y -- array containing state variables [realtype, input]
274  * FY -- array containing state derivatives [realtype, input]
275  * N -- number of matrix rows/columns in Jacobian [int, input]
276  * NNZ -- allocated length of nonzero storage [int, input]
277  * JDATA -- nonzero values in Jacobian
278  * [realtype of length NNZ, output]
279  * JRVALS -- row [or column] indices for each nonzero in Jacobian
280  * [int of length NNZ, output]
281  * JCPTRS -- pointers to each Jacobian column [or row] in preceding arrays
282  * [int of length N+1, output]
283  * H -- current step size [realtype, input]
284  * IPAR -- array containing integer user data that was passed to
285  * FCVMALLOC [long int, input]
286  * RPAR -- array containing real user data that was passed to
287  * FCVMALLOC [realtype, input]
288  * WK* -- array containing temporary workspace of same size as Y
289  * [realtype, input]
290  * IER -- return flag [int, output]:
291  * 0 if successful,
292  * >0 if a recoverable error occurred,
293  * <0 if an unrecoverable error ocurred.
294  *
295  * NOTE: this may ONLY be used if SUNDIALS has been configured with
296  * long int set to 64-bit integers.
297  *
298  * (2) Optional user-supplied Jacobian-vector product setup routine:
299  * FCVJTSETUP
300  *
301  * As an option when using the CVLS linear solver interface with a
302  * matrix-free linear solver, the user may supply a routine that computes
303  * the product of the system Jacobian J = dfi(t,y)/dy and a given vector v,
304  * as well as a routine to set up any user data structures in preparation
305  * for the matrix-vector product. If a 'setup' routine is supplied, it
306  * must have the following form:
307  *
308  * SUBROUTINE FCVJTSETUP(T, Y, FY, H, IPAR, RPAR, IER)
309  *
310  * Typically this routine will use only T and Y. It must perform any
311  * relevant preparations for subsequent calls to the user-provided
312  * FCVJTIMES routine (see below).
313  *
314  * The arguments are:
315  * T -- current time [realtype, input]
316  * Y -- array containing state variables [realtype, input]
317  * FY -- array containing state derivatives [realtype, input]
318  * H -- current step size [realtype, input]
319  * IPAR -- array containing integer user data that was passed to
320  * FCVMALLOC [long int, input]
321  * RPAR -- array containing real user data that was passed to
322  * FCVMALLOC [realtype, input]
323  * IER -- return flag [int, output]:
324  * 0 if successful,
325  * nonzero if an error.
326  *
327  * (2) Optional user-supplied Jacobian-vector product routine: FCVJTIMES
328  *
329  * As an option when using the SP* linear solver, the user may supply
330  * a routine that computes the product of the system Jacobian J = df/dy and
331  * a given vector v. If supplied, it must have the following form:
332  *
333  * SUBROUTINE FCVJTIMES (V, FJV, T, Y, FY, H, IPAR, RPAR, WORK, IER)
334  *
335  * Typically this routine will use only NEQ, T, Y, V, and FJV. It must
336  * compute the product vector Jv where the vector v is stored in V, and store
337  * the product in FJV.
338  *
339  * The arguments are:
340  * V -- array containing vector to multiply [realtype, input]
341  * JV -- array containing product vector [realtype, output]
342  * T -- current time [realtype, input]
343  * Y -- array containing state variables [realtype, input]
344  * FY -- array containing state derivatives [realtype, input]
345  * H -- current step size [realtype, input]
346  * IPAR -- array containing integer user data that was passed to
347  * FCVMALLOC [long int, input]
348  * RPAR -- array containing real user data that was passed to
349  * FCVMALLOC [realtype, input]
350  * WORK -- array containing temporary workspace of same size as Y
351  * [realtype, input]
352  * IER -- return flag [int, output]:
353  * 0 if successful,
354  * nonzero if an error.
355  *
356  * (3) Optional user-supplied preconditioner setup/solve routines: FCVPSET
357  * and FCVPSOL
358  *
359  * As an option when using the CVLS linear solver interface and an
360  * iterative linear solver, the user may supply routines to setup and
361  * apply the preconditioner. If supplied, these must have the
362  * following form:
363  *
364  * SUBROUTINE FCVPSET(T,Y,FY,JOK,JCUR,GAMMA,H,IPAR,RPAR,IER)
365  *
366  * This routine must set up the preconditioner P to be used in the
367  * subsequent call to FCVPSOL. The preconditioner (or the product of
368  * the left and right preconditioners if using both) should be an
369  * approximation to the matrix A = I - GAMMA*J (J = Jacobian),
370  *
371  * The arguments are:
372  * T = current time [realtype, input]
373  * Y = current state variable array [realtype, input]
374  * FY = current state variable derivative array [realtype, input]
375  * JOK = flag indicating whether Jacobian-related data needs to be
376  * recomputed [int, input]:
377  * 0 = recompute,
378  * 1 = reuse with the current value of GAMMA
379  * JCUR = return flag to denote if Jacobian data was recomputed
380  * [realtype, output], 1=yes, 0=no
381  * GAMMA = Jacobian scaling factor [realtype, input]
382  * H = current time step [realtype, input]
383  * IPAR = array of user integer data [long int, input/output]
384  * RPAR = array with user real data [realtype, input/output]
385  * IER = return completion flag [int, output]:
386  * 0 = SUCCESS,
387  * >0 = recoverable failure
388  * <0 = non-recoverable failure
389  *
390  * The user-supplied routine FCVPSOL must have the form:
391  *
392  * SUBROUTINE FCVPSOL(T,Y,FY,R,Z,GAMMA,DELTA,LR,IPAR,RPAR,IER)
393  *
394  * Typically this routine will use only T, Y, GAMMA, R, LR, and Z. It
395  * must solve the preconditioner linear system Pz = r. The preconditioner
396  * (or the product of the left and right preconditioners if both are
397  * nontrivial) should be an approximation to the matrix I - GAMMA*J
398  * (J = Jacobian).
399  *
400  * The arguments are:
401  * T = current time [realtype, input]
402  * Y = current state variable array [realtype, input]
403  * FY = current state variable derivative array [realtype, input]
404  * R = right-hand side array [realtype, input]
405  * Z = solution array [realtype, output]
406  * GAMMA = Jacobian scaling factor [realtype, input]
407  * DELTA = desired residual tolerance [realtype, input]
408  * LR = flag denoting to solve the right or left preconditioner system
409  * 1 = left preconditioner
410  * 2 = right preconditioner
411  * IPAR = array of user integer data [long int, input/output]
412  * RPAR = array with user real data [realtype, input/output]
413  * IER = return completion flag [int, output]:
414  * 0 = SUCCESS,
415  * >0 = recoverable failure
416  * <0 = non-recoverable failure
417  *
418  * (4) Optional user-supplied error weight vector routine: FCVEWT
419  *
420  * As an option to providing the relative and absolute tolerances, the user
421  * may supply a routine that computes the weights used in the WRMS norms.
422  * If supplied, it must have the following form:
423  *
424  * SUBROUTINE FCVEWT (Y, EWT, IPAR, RPAR, IER)
425  *
426  * It must store the error weights in EWT, given the current solution vector Y.
427  *
428  * The arguments are:
429  * Y -- array containing state variables [realtype, input]
430  * EWT -- array containing the error weight vector [realtype, output]
431  * IPAR -- array containing integer user data that was passed to
432  * FCVMALLOC [long int, input]
433  * RPAR -- array containing real user data that was passed to
434  * FCVMALLOC [realtype, input]
435  * IER -- return flag [int, output]:
436  * 0 if successful,
437  * nonzero if an error.
438  *
439  *
440  * -----------------------------------------------------------------------------
441  *
442  * (5) Initialization: FNVINITS / FNVINITP / FNVINITOMP / FNVINITPTS,
443  * FSUNBANDMATINIT / FSUNDENSEMATINIT /
444  * FSUNSPARSEMATINIT,
445  * FSUNBANDLINSOLINIT / FSUNDENSELINSOLINIT /
446  * FSUNKLUINIT / FSUNKLUREINIT /
447  * FSUNKLUSETORDERING / FSUNLAPACKBANDINIT /
448  * FSUNLAPACKDENSEINIT / FSUNPCGINIT /
449  * FSUNSPBCGSINIT / FSUNSPFGMRINIT / FSUNSPGMRINIT /
450  * FSUNSPTFQMRINIT / FSUNSUPERLUMTINIT /
451  * FSUNSUPERLUMTSETORDERING,
452  * FCVMALLOC,
453  * FCVLSINIT,
454  * FCVREINIT
455  *
456  * NOTE: the initialization order is important! It *must* proceed as
457  * shown: vector, matrix (if used), linear solver (if used), CVode,
458  * CVLs, reinit.
459  *
460  * (5.1s) To initialize the a vector specification for storing the solution
461  * data, the user must make one of the following calls:
462  *
463  * (serial)
464  * CALL FNVINITS(1, NEQ, IER)
465  * (MPI parallel)
466  * CALL FNVINITP(COMM, 1, NLOCAL, NGLOBAL, IER)
467  * (OpenMP threaded)
468  * CALL FNVINITOMP(1, NEQ, NUM_THREADS, IER)
469  * (PThreads threaded)
470  * CALL FNVINITPTS(1, NEQ, NUM_THREADS, IER)
471  *
472  * In each of these, one argument is an int containing the CVODE solver
473  * ID (1).
474  *
475  * The other arguments are:
476  * NEQ = size of vectors [long int, input]
477  * COMM = the MPI communicator [int, input]
478  * NLOCAL = local size of vectors on this processor
479  * [long int, input]
480  * NGLOBAL = the system size, and the global size of vectors (the sum
481  * of all values of NLOCAL) [long int, input]
482  * NUM_THREADS = number of threads
483  * IER = return completion flag [int, output]:
484  * 0 = success,
485  * -1 = failure.
486  *
487  * (5.2) To initialize a band/dense/sparse matrix structure for
488  * storing the system Jacobian and for use within a direct linear solver,
489  * the user must make one of the following calls:
490  *
491  * CALL FSUNBANDMATINIT(1, N, MU, ML, SMU, IER)
492  * CALL FSUNDENSEMATINIT(1, M, N, IER)
493  * CALL FSUNSPARSEMATINIT(1, M, N, NNZ, SPARSETYPE, IER)
494  *
495  * In each of these, one argument is an int containing the CVODE solver
496  * ID (1).
497  *
498  * The other arguments are:
499  *
500  * M = the number of rows of the matrix [long int, input]
501  * N = the number of columns of the matrix [long int, input]
502  * MU = the number of upper bands (diagonal not included) in a banded
503  * matrix [long int, input]
504  * ML = the number of lower bands (diagonal not included) in a banded
505  * matrix [long int, input]
506  * SMU = the number of upper bands to store (diagonal not included)
507  * for factorization of a banded matrix [long int, input]
508  * NNZ = the storage size (upper bound on the number of nonzeros) for
509  * a sparse matrix [long int, input]
510  * SPARSETYPE = integer denoting use of CSC (0) vs CSR (1) storage
511  * for a sparse matrix [int, input]
512  * IER = return completion flag [int, output]:
513  * 0 = success,
514  * -1 = failure.
515  *
516  * (5.3) To initialize a linear solver structure for solving linear systems
517  * arising from implicit treatment of the IVP, the user must make
518  * one of the following calls:
519  *
520  * CALL FSUNBANDLINSOLINIT(1, IER)
521  * CALL FSUNDENSELINSOLINIT(1, IER)
522  * CALL FSUNKLUINIT(1, IER)
523  * CALL FSUNLAPACKBANDINIT(1, IER)
524  * CALL FSUNLAPACKDENSEINIT(1, IER)
525  * CALL FSUNPCGINIT(1, PRETYPE, MAXL, IER)
526  * CALL FSUNSPBCGSINIT(1, PRETYPE, MAXL, IER)
527  * CALL FSUNSPFGMRINIT(1, PRETYPE, MAXL, IER)
528  * CALL FSUNSPGMRINIT(1, PRETYPE, MAXL, IER)
529  * CALL FSUNSPTFQMRINIT(1, PRETYPE, MAXL, IER)
530  * CALL FSUNSUPERLUMTINIT(1, NUM_THREADS, IER)
531  *
532  * Or once these have been initialized, their solver parameters may be
533  * modified via calls to the functions
534  *
535  * CALL FSUNKLUSETORDERING(1, ORD_CHOICE, IER)
536  * CALL FSUNSUPERLUMTSETORDERING(1, ORD_CHOICE, IER)
537  *
538  * CALL FSUNPCGSETPRECTYPE(1, PRETYPE, IER)
539  * CALL FSUNPCGSETMAXL(1, MAXL, IER)
540  * CALL FSUNSPBCGSSETPRECTYPE(1, PRETYPE, IER)
541  * CALL FSUNSPBCGSSETMAXL(1, MAXL, IER)
542  * CALL FSUNSPFGMRSETGSTYPE(1, GSTYPE, IER)
543  * CALL FSUNSPFGMRSETPRECTYPE(1, PRETYPE, IER)
544  * CALL FSUNSPGMRSETGSTYPE(1, GSTYPE, IER)
545  * CALL FSUNSPGMRSETPRECTYPE(1, PRETYPE, IER)
546  * CALL FSUNSPTFQMRSETPRECTYPE(1, PRETYPE, IER)
547  * CALL FSUNSPTFQMRSETMAXL(1, MAXL, IER)
548  *
549  * In all of the above, one argument is an int containing the CVODE solver
550  * ID (1).
551  *
552  * The other arguments are:
553  *
554  * NNZ = the storage size (upper bound on the number of nonzeros) for
555  * a sparse matrix [long int, input]
556  * ORD_CHOICE = integer denoting ordering choice (see
557  * SUNKLUSetOrdering and SUNSuperLUMTSetOrdering documentation
558  * for details) [int, input]
559  * PRETYPE = type of preconditioning to perform (0=none, 1=left,
560  * 2=right, 3=both) [int, input]
561  * MAXL = maximum Krylov subspace dimension [int, input]
562  * GSTYPE = choice of Gram-Schmidt orthogonalization algorithm
563  * (0=modified, 1=classical) [int, input]
564  * IER = return completion flag [int, output]:
565  * 0 = success,
566  * -1 = failure.
567  *
568  *
569  * (5.4) To set various problem and solution parameters and allocate
570  * internal memory, make the following call:
571  *
572  * CALL FCVMALLOC(T0, Y0, METH, IATOL, RTOL, ATOL,
573  * 1 IOUT, ROUT, IPAR, RPAR, IER)
574  *
575  * The arguments are:
576  * T0 = initial value of t [realtype, input]
577  * Y0 = array of initial conditions [realtype, input]
578  * METH = flag denoting basic integration method [int, input]:
579  * 1 = Adams (nonstiff),
580  * 2 = BDF (stiff)
581  * IATOL = flag denoting type for absolute tolerance ATOL [int, input]:
582  * 1 = scalar,
583  * 2 = array.
584  * 3 = user-supplied function; the user must supply a routine
585  * FCVEWT to compute the error weight vector.
586  * RTOL = scalar relative tolerance [realtype, input]
587  * ATOL = scalar or array absolute tolerance [realtype, input]
588  * IOUT = array of length 21 for integer optional outputs
589  * [long int, output]
590  * ROUT = array of length 6 for real optional outputs [realtype, output]
591  * IPAR = array with user integer data [long int, input/output]
592  * RPAR = array with user real data [realtype, input/output]
593  * IER = return completion flag [int, output]:
594  * 0 = SUCCESS,
595  * -1 = failure (see printed message for failure details).
596  *
597  * The user data arrays IPAR and RPAR are passed unmodified to all subsequent
598  * calls to user-provided routines. Modifications to either array inside a
599  * user-provided routine will be propagated. Using these two arrays, the user
600  * can dispense with Common blocks to pass data betwen user-provided routines.
601  *
602  * The optional outputs are:
603  * LENRW = IOUT( 1) from CVodeGetWorkSpace
604  * LENIW = IOUT( 2) from CVodeGetWorkSpace
605  * NST = IOUT( 3) from CVodeGetNumSteps
606  * NFE = IOUT( 4) from CVodeGetNumRhsEvals
607  * NETF = IOUT( 5) from CVodeGetNumErrTestFails
608  * NCFN = IOUT( 6) from CVodeGetNumNonlinSolvConvFails
609  * NNI = IOUT( 7) from CVodeGetNumNonlinSolvIters
610  * NSETUPS = IOUT( 8) from CVodeGetNumLinSolvSetups
611  * QU = IOUT( 9) from CVodeGetLastOrder
612  * QCUR = IOUT(10) from CVodeGetCurrentOrder
613  * NOR = IOUT(11) from CVodeGetNumStabLimOrderReds
614  * NGE = IOUT(12) from CVodeGetNumGEvals
615  *
616  * H0U = ROUT( 1) from CVodeGetActualInitStep
617  * HU = ROUT( 2) from CVodeGetLastStep
618  * HCUR = ROUT( 3) from CVodeGetCurrentStep
619  * TCUR = ROUT( 4) from CVodeGetCurrentTime
620  * TOLSF = ROUT( 5) from CVodeGetTolScaleFactor
621  * UROUND = ROUT( 6) from UNIT_ROUNDOFF
622  * See the CVODE manual for details.
623  *
624  * (5.5) If a linear solver was created in step (5.3) then it must be
625  * attached to CVode. If the user called any one of FSUNBANDLINSOLINIT,
626  * FSUNDENSELINSOLINIT, FSUNKLUINIT, FSUNLAPACKBANDINIT,
627  * FSUNLAPACKDENSEINIT, FSUNSUPERLUMTINIT, FSUNPCGINIT,
628  * FSUNSPBCGSINIT, FSUNSPFGMRINIT, FSUNSPGMRINIT, or FSUNSPTFQMRINIT,
629  * then this must be attached to the CVLS interface using the command:
630  *
631  * CALL FCVLSINIT(IER)
632  *
633  * The arguments are:
634  * IER = return completion flag [int, output]:
635  * 0 = SUCCESS,
636  * -1 = failure (see printed message for failure details).
637  *
638  * (5.5) If the user instead wishes to use a diagonal approximate Jacobian for
639  * solving the Newton systems, then it must be created and attached to CVode.
640  * This choice is appropriate when the Jacobian can be well approximated by
641  * a diagonal matrix. The user must make the call:
642  * CALL FCVDIAG(IER)
643  *
644  * The arguments are:
645  * IER = return completion flag [int, output]:
646  * 0 = SUCCESS,
647  * -1 = failure (see printed message for failure details).
648  *
649  * (5.6) If the user program includes the FCVEWT routine for the evaluation
650  * of the error weights, the following call must be made
651  *
652  * CALL FCVEWTSET(FLAG, IER)
653  *
654  * with FLAG = 1 to specify that FCVEWT is provided and
655  * should be used; FLAG = 0 resets to the default EWT formulation.
656  * The return flag IER is 0 if successful, and nonzero otherwise.
657  *
658  * (5.7) If the user program includes the FCVBJAC routine for the
659  * evaluation of the band approximation to the Jacobian, then following
660  * the call to FCVLSINIT, the following call must be made
661  *
662  * CALL FCVBANDSETJAC(FLAG, IER)
663  *
664  * with the int FLAG=1 to specify that FCVBJAC is provided and should be
665  * used; FLAG=0 specifies a reset to the internal finite difference
666  * Jacobian approximation. The int return flag IER=0 if successful,
667  * nonzero otherwise.
668  *
669  * If the user program includes the FCVDJAC routine for the evaluation
670  * of the dense approximation to the Jacobian, then after the call to
671  * FCVLSINIT, the following call must be made
672  *
673  * CALL FCVDENSESETJAC(FLAG, IER)
674  *
675  * with the int FLAG=1 to specify that FCVDJAC is provided and should be
676  * used; FLAG=0 specifies a reset to the internal finite difference
677  * Jacobian approximation. The int return flag IER=0 if successful, and
678  * nonzero otherwise.
679  *
680  * When using a sparse matrix and linear solver the user must provide the
681  * FCVSPJAC routine for the evaluation of the sparse approximation to
682  * the Jacobian. To indicate that this routine has been provided, after
683  * the call to FCVLSINIT, the following call must be made
684  *
685  * CALL FCVSPARSESETJAC(IER)
686  *
687  * The int return flag IER=0 if successful, and nonzero otherwise.
688  *
689  * (5.8) If the user program includes the FCVJTSETUP and FCVJTIMES
690  * routines for setup of a Jacobian-times-vector product (for use with
691  * the CVLS interface), then after creating the CVLS interface,
692  * the following call must be made:
693  *
694  * CALL FCVLSSETJAC(FLAG, IER)
695  *
696  * with the int FLAG=1 to specify that FCVJTSETUP and FCVJTIMES are
697  * provided and should be used; FLAG=0 specifies a reset to the internal
698  * finite difference approximation to this product). The int return
699  * flag IER=0 if successful, and nonzero otherwise.
700  *
701  * (5.9) If the user program includes the FCVPSET and FCVPSOL routines
702  * for supplying a preconditioner to an iterative linear solver, then
703  * after creating the CVLS interface, the following call must be made
704  *
705  * CALL FCVLSSETPREC(FLAG, IER)
706  *
707  * with the int FLAG=1. If FLAG=0 then preconditioning with these
708  * routines will be disabled. The return flag IER=0 if successful,
709  * nonzero otherwise.
710  *
711  * (5.10) If the user wishes to use one of CVode's built-in preconditioning
712  * modules, FCVBP or FCVBBD, then that should be initialized after
713  * creating the CVLS interface using one of the calls
714  *
715  * CALL FCVBPINIT(NEQ, MU, ML, IER)
716  * CALL FCVBBDINIT(NLOCAL, MUDQ, MLDQ, MU, ML, DQRELY, IER)
717  *
718  * Detailed explanation of the inputs to these functions, as well as any
719  * requirements of user-supplied functions on which these preconditioning
720  * modules rely, may be found in the header files for each module,
721  * fcvbp.h or fcvbbd.h, respectively.
722  *
723  *
724  *
725  * (5.11) To re-initialize the CVODE solver for the solution of a new problem
726  * of the same size as one already solved, make the following call:
727  *
728  * CALL FCVREINIT(T0, Y0, IATOL, RTOL, ATOL, IER)
729  *
730  * The arguments have the same names and meanings as those of FCVMALLOC,
731  * except that METH has been omitted from the argument list
732  * (being unchanged for the new problem).
733  * FCVREINIT performs the same initializations as FCVMALLOC, but does no memory
734  * allocation, using instead the existing internal memory created by the
735  * previous FCVMALLOC call. The subsequent calls to
736  * attach the linear system solver is only needed if that object has been re-created.
737  *
738  * (5.12) The SUNKLU solver will reuse much of the factorization information
739  * from one solve to the next. If at any time the user wants to force a
740  * full refactorization or if the number of nonzeros in the Jacobian
741  * matrix changes, the user should make the call
742  *
743  * CALL FSUNKLUREINIT(4, NNZ, REINIT_TYPE, IER)
744  *
745  * The arguments are:
746  * NNZ = the maximum number of nonzeros [int; input]
747  * REINIT_TYPE = 1 or 2. For a value of 1, the matrix will be
748  * destroyed and a new one will be allocated with NNZ nonzeros.
749  * For a value of 2, only symbolic and numeric factorizations will
750  * be completed.
751  *
752  * (5.13) To set various integer optional inputs, make the folowing call:
753  *
754  * CALL FCVSETIIN(KEY, VALUE, IER)
755  *
756  * to set the integer value VAL to the optional input specified by the
757  * quoted character string KEY. VALUE must be a Fortran integer of size
758  * commensurate with a C "long int".
759  * KEY must be one of the following: MAX_ORD, MAX_NSTEPS, MAX_ERRFAIL,
760  * MAX_NITERS, MAX_CONVFAIL, HNIL_WARNS, STAB_LIM. The int return flag
761  * IER is 0 if successful, and <0 otherwise.
762  *
763  * (5.14) To set various real optional inputs, make the folowing call:
764  *
765  * CALL FCVSETRIN(KEY, VALUE, IER)
766  *
767  * to set the real value VAL to the optional input specified by the
768  * quoted character string KEY. VALUE must be a Fortran real-valued
769  * number of size commensurate with the SUNDIALS "realtype". KEY must
770  * be one of the following: INIT_STEP, MAX_STEP, MIN_STEP, STOP_TIME,
771  * NLCONV_COEF. The int return flag IER is 0 if successful, and <0 otherwise.
772  *
773  * (5.15) To set the vector of constraints, make the following call:
774  *
775  * CALL CVSETVIN(KEY, ARRAY, IER)
776  *
777  * where ARRAY is an array of realtype and the quoted character string
778  * KEY is CONSTR_VEC. The int return flag IER is 0 if successful, and
779  * nonzero otherwise.
780  *
781  * -----------------------------------------------------------------------------
782  *
783  * (6) Optional outputs from CVLS linear solvers (stored in the
784  * IOUT array that was passed to FCVMALLOC)
785  *
786  * Optional outputs specific to the CVLS interface:
787  * LENRWLS = IOUT(13) from CVodeGetLinWorkSpace (realtype space)
788  * LENIWLS = IOUT(14) from CVodeGetLinWorkSpace (integer space)
789  * LSTF = IOUT(15) from CVodeGetLastLinFlag
790  * NFELS = IOUT(16) from CVodeGetNumLinRhsEvals
791  * NJE = IOUT(17) from CVodeGetNumJacEvals
792  * NJTS = IOUT(18) from CVodeGetNumJTSetupEvals
793  * NJTV = IOUT(19) from CVodeGetNumJtimesEvals
794  * NPE = IOUT(20) from CVodeGetNumPrecEvals
795  * NPS = IOUT(21) from CVodeGetNumPrecSolves
796  * NLI = IOUT(22) from CVodeGetNumLinIters
797  * NCFL = IOUT(23) from CVodeGetNumLinConvFails
798  *
799  * Optional outputs specific to the DIAG case are:
800  * LENRWLS = IOUT(13) from CVDiagGetWorkSpace
801  * LENIWLS = IOUT(14) from CVDiagGetWorkSpace
802  * LSTF = IOUT(15) from CVDiagGetLastFlag
803  * NFELS = IOUT(16) from CVDiagGetNumRhsEvals
804  *
805  * See the CVODE manual for more detailed descriptions of any of the
806  * above.
807  *
808  * -----------------------------------------------------------------------------
809  *
810  * (7) The integrator: FCVODE
811  *
812  * Carrying out the integration is accomplished by making calls as follows:
813  *
814  * CALL FCVODE (TOUT, T, Y, ITASK, IER)
815  *
816  * The arguments are:
817  * TOUT = next value of t at which a solution is desired [realtype, input]
818  * T = value of t reached by the solver on output [realtype, output]
819  * Y = array containing the computed solution on output [realtype, output]
820  * ITASK = task indicator [int, input]:
821  * 1 = normal mode (overshoot TOUT and interpolate)
822  * 2 = one-step mode (return after each internal step taken)
823  * 3 = normal tstop mode (like 1, but integration never proceeds past
824  * TSTOP, which must be specified through a call to FCVSETRIN
825  * using the key 'STOP_TIME')
826  * 4 = one step tstop (like 2, but integration never goes past TSTOP)
827  * IER = completion flag [int, output]:
828  * 0 = success,
829  * 1 = tstop return,
830  * 2 = root return,
831  * values -1 ... -10 are various failure modes (see CVODE manual).
832  * The current values of the optional outputs are available in IOUT and ROUT.
833  *
834  * -----------------------------------------------------------------------------
835  *
836  * (8) Computing solution derivatives: FCVDKY
837  *
838  * To obtain a derivative of the solution, of order up to the current method
839  * order, make the following call:
840  *
841  * CALL FCVDKY (T, K, DKY, IER)
842  *
843  * The arguments are:
844  * T = value of t at which solution derivative is desired, in
845  * [TCUR-HU,TCUR], [realtype, input].
846  * K = derivative order (0 .le. K .le. QU) [int, input]
847  * DKY = array containing computed K-th derivative of y [realtype, output]
848  * IER = return flag [int, output]: = 0 for success, <0 for illegal argument
849  *
850  * -----------------------------------------------------------------------------
851  *
852  * (9) Get the current error weight vector: FCVGETERRWEIGHTS
853  *
854  * To obtain the current error weight vector, make the following call:
855  *
856  * CALL FCVGETERRWEIGHTS(EWT, IER)
857  *
858  * The arguments are:
859  * EWT = array containing the error weight vector [realtype, output]
860  * IER = return flag [int, output]: 0=success, nonzero if an error.
861  *
862  * -----------------------------------------------------------------------------
863  *
864  * (10) Memory freeing: FCVFREE
865  *
866  * To free the internal memory created by the calls to FCVMALLOC,
867  * FCVLSINIT and FNVINIT*, make the call
868  *
869  * CALL FCVFREE
870  *
871  * =============================================================================
872  */
