15  * This is the header file for a generic linear solver package.
16  * It defines the SUNLinearSolver structure (_generic_SUNLinearSolver)
17  * which contains the following fields:
18  * - an implementation-dependent 'content' field which contains
19  * any internal data required by the solver
20  * - an 'ops' filed which contains a structure listing operations
21  * acting on/by such solvers
22  *
23  * We consider both direct linear solvers and iterative linear solvers
24  * as available implementations of this package. Furthermore, iterative
25  * linear solvers can either use a matrix or be matrix-free. As a
26  * result of these different solver characteristics, some of the
27  * routines are applicable only to some types of linear solver.
28  * -----------------------------------------------------------------
29  * This header file contains:
30  * - enumeration constants for all SUNDIALS-defined linear solver
31  * types, as well as a generic type for user-supplied linear
32  * solver types,
33  * - type declarations for the _generic_SUNLinearSolver and
34  * _generic_SUNLinearSolver_Ops structures, as well as references
35  * to pointers to such structures (SUNLinearSolver),
36  * - prototypes for the linear solver functions which operate
37  * on/by SUNLinearSolver objects, and
38  * - return codes for SUNLinearSolver objects.
39  * -----------------------------------------------------------------
40  * At a minimum, a particular implementation of a SUNLinearSolver must
41  * do the following:
42  * - specify the 'content' field of SUNLinearSolver,
43  * - implement the operations on/by those SUNLinearSolver objects,
44  * - provide a constructor routine for new SUNLinearSolver objects
45  *
46  * Additionally, a SUNLinearSolver implementation may provide the
47  * following:
48  * - "Set" routines to control solver-specific parameters/options
49  * - "Get" routines to access solver-specific performance metrics
50  * -----------------------------------------------------------------*/
